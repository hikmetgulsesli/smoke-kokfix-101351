# Sayaç Uygulaması PRD — smoke-kokfix-101351

## 1. Genel Bakış

smoke-kokfix-101351, web tabanlı, tam davranışlı bir sayaç uygulamasıdır. Kullanıcılar ana ekrandan sayaç değerini artırıp azaltabilir, yeni kayıtlar oluşturabilir, geçmiş kayıtları inceleyip silebilir ve uygulama tercihlerini ayarlar panelinden yönetebilir. Tasarım Stitch referansına sadık kalınarak koyu tema ve responsive web deneyimi sunar. Hedef kitle: günlük sayaç ihtiyacı olan bireysel kullanıcılar; arayüz tamamen Türkçedir.

## 2. Hedefler

1. Hızlı ve anlaşılır bir sayaç deneyimi sunmak; ana ekranda sayaç değeri ve temel kontroller anlık erişilebilir olsun.
2. Kullanıcıların sayaç anlarını başlık ve notla kaydetmesini sağlamak; kayıtlar localStorage’da kalıcı olsun.
3. Geçmiş kayıtları listeleyip yönetmek; tek kayıt silme ve tümünü temizleme işlevsel olsun.
4. Koyu tema ve görsel tercihleri ayarlar panelinden yönetmek; tercihler oturumlar arasında korunsun.
5. Erişilebilir ve responsive bir arayüz inşa etmek; klavye navigasyonu, ekran okuyucu desteği ve mobil uyumluluk sağlansın.
6. Stitch tasarım diline sadık kalarak tutarlı, sade ve modern bir görünüm elde etmek.

## 3. Tech Stack Detayı

- **Framework:** React 18.3 (Functional Components + Hooks)
- **Build:** Vite 5.x (HMR, hızlı geliştirme, optimize üretim derlemesi)
- **Dil:** TypeScript 5.x (strict mode, tip güvenliği)
- **Styling:** Tailwind CSS 3.4 (utility-first, koyu tema varyantları)
- **State:** React useState/useReducer (yerel state yönetimi; harici state kütüphanesine gerek yok)
- **Storage:** localStorage (sayacın anlık değeri, kayıt listesi, tema tercihi)
- **Routing:** React Router DOM 6.x (SPA, hash-based veya memory router; sayfalar arası geçiş < 100 ms)
- **Icon Library:** Lucide React (ikonlar aria-label/title ile erişilebilir)
- **Araçlar:** ESLint + Prettier, Vite PWA plugin (opsiyonel ileri aşama)

## 4. Fonksiyonel Gereksinimler

### 4.1 Ana Ekran Sayaç
- Ekran ortasında büyük sayaç değeri görüntülenir.
- Butonlar: "Artır" (+1), "Azalt" (-1), "Sıfırla" (0).
- Sayaç değeri localStorage’da `counter_value` anahtarıyla saklanır; sayfa yenilendiğinde değer korunur.
- Sayaç negatif değerlere inebilir.
- Butonlarda hover ve active durumları Tailwind ile sağlanır.

### 4.2 Üst Bar / Tab Bar İkon Kontrolleri
- Sol/orta/sağ düzende üç ikon kontrolü: Ekle/Yeni (Plus), Geçmiş/Kayıtlar (History/ClipboardList), Ayarlar (Settings).
- Her ikon yalnızca ikondan oluşur; `aria-label` ve `title` nitelikleri kesinlikle dolu olmalıdır.
- `onClick` boş fonksiyon (`() => {}`) olamaz; gerçek handler atanmalıdır.
- Aktif sekme/panel görsel olarak vurgulanır (örneğin alt çizgi veya arka plan değişimi).

### 4.3 Ekle/Yeni Kayıt Modalı
- "Ekle" ikonuna tıklandığında modal açılır.
- Modal içeriği:
  - Başlık inputu (metin, zorunlu, max 100 karakter, placeholder: "Kayıt başlığı…")
  - Not textarea (metin, opsiyonel, max 500 karakter, placeholder: "Not ekleyin…")
  - Butonlar: "İptal" (modalı kapatır, veri sıfırlanır), "Kapat" (aynı davranış), "Oluştur" (kaydı kaydeder).
- Validasyon: başlık boşsa "Lütfen başlık girin" mesajı gösterilir.
- Başarı durumunda modal kapanır, kayıt localStorage’da `counter_records` dizisine eklenir; her kayıt `{ id, title, note, value, createdAt }` yapısında saklanır.

### 4.4 Geçmiş/Kayıtlar Ekranı
- "Geçmiş" ikonuna tıklandığında kayıtlar ekranı/paneli açılır.
- Kayıtlar yeniden eskiye sıralanır (en yeni üstte).
- Her kayıt kartında: başlık, not (varsa), o anki sayaç değeri, tarih.
- İşlemler:
  - "Sil" butonu (tek kayıt); onay sorusu: "Bu kaydı silmek istediğinize emin misiniz?"
  - "Tümünü Temizle" butonu; onay sorusu: "Tüm kayıtları silmek istediğinize emin misiniz? Bu işlem geri alınamaz."
- Boş durum: "Henüz kayıt oluşturulmadı." mesajı ve yönlendirme.

### 4.5 Ayarlar Ekranı
- "Ayarlar" ikonuna tıklandığında ayarlar paneli/sayfası açılır.
- Tercihler:
  - Tema: Koyu / Sistem (varsayılan: Koyu)
  - Sayaç adım boyutu (opsiyonel ileri aşama): 1, 5, 10
- Tercihler `counter_preferences` anahtarıyla localStorage’da saklanır.
- Ayarlar değiştiğinde anlık uygulanır; sayfa yenilendiğinde korunur.

## 5. Veri Modeli

### localStorage Schema

```
counter_value: number
  └─ Ana sayaç değeri

counter_records: Array<Record>
  Record: {
    id: string (uuid v4)
    title: string (max 100)
    note: string (max 500)
    value: number (kaydedildiği anki sayaç değeri)
    createdAt: string (ISO 8601)
  }

counter_preferences: {
  theme: "dark" | "system"
  stepSize: 1 | 5 | 10
}
```

- Tüm veriler `JSON.stringify` ile yazılır, `JSON.parse` ile okunur.
- Okuma hatasında (bozuk JSON): varsayılan değerler döndürülür ve storage temizlenir.

## 7. Non-Functional Requirements

### 7.1 Performans
- İlk yükleme < 2 saniye (Vite + lazy load yok; tek bundle)
- Sayfa/panel geçişi < 100 ms (client-side routing, React state)
- Bundle size hedefi: < 200 KB gzipped (Vite tree-shaking ile)

### 7.2 Erişilebilirlik (WCAG 2.1 AA)
- Klavye navigasyonu: Tab ile tüm kontrollere erişim, Enter/Space ile aktivasyon.
- Screen reader: Tüm ikon-only butonlar `aria-label` + `title` içerir; modal `role="dialog"`, `aria-modal="true"`.
- Kontrast: Metin ≥ 4.5:1, büyük metin ≥ 3:1 (Tailwind koyu tema renkleri ile otomatik sağlanır).
- Focus state: Tüm interaktif elemanlarda `focus-visible:ring-2 focus-visible:ring-violet-500`.
- Touch hedefi: Butonlar ve ikonlar minimum 44 × 44 px.

### 7.3 Tarayıcı Desteği
- Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- iOS Safari 14+, Android Chrome 90+
- Responsive: 320 px (mobil) → 1920 px+ (masaüstü)
